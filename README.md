# LizardW2Hack

Проект для модификации Unity игр (**Il2CPP + Mono**). Из коробки ориентирован на Humans Are Not That Against LizardWoman 2, но **список хуков задаётся текстовым конфигом** и может быть переписан под другую игру.

## Описание

Нативная DLL (`Lizard.dll`) загружается в процесс игры через **Unity Doorstop** и managed-bootstrap. Перехват методов делается через **MinHook**.

- Для **Il2CPP**: адреса методов находятся резолвером по `global-metadata.dat` + `GameAssembly.dll`.
- Для **Mono**: адреса и сигнатуры методов определяются в рантайме через Mono API.

> На текущем этапе Mono-инжект выполняется **вручную** (внешним инжектором), после чего проект работает в том же формате, что и в Il2CPP: те же вкладки, конфиг хуков, переключение ON/OFF и runtime-поиск методов.

Поведение каждого хука (какой метод, что логировать, что возвращать, включён ли при старте) задаётся в **`LizardHooks.txt`** в формате, похожем на объявление функции на C++.

## Поддерживаемые версии Unity / IL2CPP

Резолвер ориентируется не на номер **Unity Editor** в интерфейсе, а на **целое число `version` в заголовке** файла `global-metadata.dat` (тот же смысл, что в [Il2CppDumper](https://github.com/Perfare/Il2CppDumper): поле `sanity == 0xFAB11BAF`, далее `int32 version`).

| `global-metadata` **version** | Что ожидает код (упрощённо) | Примерный диапазон Unity (не догма) |
|-------------------------------|------------------------------|-------------------------------------|
| **≤ 24** | Старые раскладки `Il2CppTypeDefinition` / `Il2CppMethodDefinition` (ветка «24») | Часть сборок **2019.x–2020.x** и старее |
| **25–30** | «Современный» тип без `byrefTypeIndex` в type-def; метод без `returnParameterToken` | Типично **2021.x–2022.x** (зависит от патча) |
| **≥ 31** | Как 25–30 для типов; у метода добавлено поле `returnParameterToken` | Начиная с ряда сборок **2022.3 LTS** (например 2022.3.35+) и **2023+ / Unity 6** |

**Фактическая поддержка в проекте:** разбор метаданных и RVA **рассчитаны на `global-metadata.dat` с version 24–31** (структуры согласованы с актуальными описаниями Il2CppDumper для этих номеров). Сборки с **metadata version 32+** в текущем виде **могут не совпасть** с раскладкой полей — потребуется доработка `il2cpp_resolver.h`.

**Как узнать версию у своей игры:** откройте `…_Data\il2cpp_data\Metadata\global-metadata.dat` в hex-редакторе: байты `AF 1B B1 FA` (little-endian `0xFAB11BAF`), следующие **4 байта** — `version` (например `1F 00 00 00` = **31**).

## Поддержка Mono

- В проекте есть runtime-режим **Mono** с детектом движка, поиском методов в загруженных сборках и установкой native-хуков для поддерживаемых bool-сигнатур.
- Текущий workflow для Mono: **запустить игру и выполнить инжект вручную** (автоинжект в репозитории не включён).
- После ручного инжекта UX такой же, как в Il2CPP: используется тот же `LizardHooks.txt`, то же меню и runtime-редактирование/перезагрузка конфигурации.

## Конфигурация хуков `LizardHooks.txt`

### Где лежит файл

- По умолчанию: **`LizardHooks.txt` в каталоге исполняемого файла игры** (рядом с `.exe`, `GetModuleHandle(NULL)`), а не рядом с `Lizard.dll`.
- Переопределение пути: переменная окружения **`LIZARD_HOOKS_FILE`** — полный путь к `.txt` (wide path, как обычный путь в Windows).

Файл читается как **UTF-8** (поддерживается BOM). Строки `// ...` считаются комментариями и вырезаются перед разбором.

### Лимиты

- До **32** хуков (слотов). Порядок в файле = порядок слотов.

### Формат блока хука

Каждый хук — блок вида **`ИмяКакВIl2Cpp$$(параметры) { тело }`**:

- Слева от **`$$`** — полное имя класса в стиле Il2Cpp: `Класс` или `Пространство.Класс`.
- Справа от **`$$`** — имя метода.
- В скобках — список параметров для **выбора сигнатуры детура** (см. ниже). Достаточно перечислить «как в примере»: обычно это указатели `DWORD*` под видом `__this`, `method`, аргументы сцен и т.д.
- В теле задаются имя для меню, стартовое включение, число параметров для резолвера, лог и возвращаемое значение.

**Сигнатура детура** выводится по **числу `DWORD*`** в списке параметров в скобках:

| `DWORD*` в скобках | Сценарий |
|--------------------|----------|
| 1 | статический метод (`BoolMethod`) |
| 2 | instance + `method` (`BoolThisMethod`) |
| 3 | три указателя (часто сцены) (`BoolThisScenes`) |
| 4 | четыре указателя (`BoolThis4`) |
| 5 | пять указателей (`BoolThis5`) |

Поле **`param N`** — число параметров **метода Il2Cpp** для `FindMethodOffsetByClassAndMethod` (как в исходнике игры, без учёта «this» в reflection API так, как ожидает резолвер в проекте). Его нужно подобрать под конкретный метод (часто совпадает с количеством «логических» аргументов в игре).

### Поля внутри `{ ... }`

| Поле | Пример | Назначение |
|------|--------|------------|
| `name "..."` | `name "Infinite coins";` | Подпись в консольном меню (UTF-8). Если нет — в меню показывается полное имя `Class$$Method`. |
| `enabled` / `enabled_on_start` / `start_enabled` | `enabled true;` | Включён ли хук **при старте** (`true` / `false`). |
| `param` или `param_count` | `param 2;` | Явное число параметров метода для резолвера (перекрывает значение по умолчанию от сигнатуры). |
| `printf("...");` | `printf("hook\\n");` | Строка при срабатывании, когда хук **включён** в меню. |
| `return true;` / `return false;` | `return true;` | Возвращаемое значение при вызове детура, если хук **включён**. |

Если хук **выключен** в меню, вызывается **оригинальная** функция игры (без подмены `return` и без `printf` из конфига).

### Пример с пояснениями

Ниже один блок «как шаблон» — что означает каждая часть.

```text
// Полное имя метода Il2Cpp: Namespace.Class$$MethodName
// В скобках — список параметров (для выбора типа детура по числу DWORD*)
Player$$isSceneOpened(DWORD* __this, DWORD* novelId, DWORD* scenario, DWORD* method) {
    name "Open any scene";   // Текст в меню
    enabled true;            // При старте игры хук включён
    param 2;                 // Il2Cpp: два параметра у метода (например novelId + scenario)
    printf("scene hook\n");  // Лог при срабатывании, если хук включён
    return true;             // Если хук включён — вернуть true из метода
}
```

Готовый файл с несколькими хуками под LizardWoman 2 см. **`LizardHooks.txt.example`** в корне репозитория — скопируйте его как **`LizardHooks.txt`** рядом с `.exe` игры.

## Консоль, меню, логотип

После инициализации выводится **баннер** (логотип UTF-8), затем открывается **главное меню** в консоли (цикл меню запускается **всегда**, даже если `LizardHooks.txt` пустой или хуки не загрузились).

Логотип брайля выводится через **`WriteConsoleW`**; консоль пытается переключить шрифт на вариант с глифами Braille (Cascadia Mono и др.). Если брайль отображается квадратиками, а при копировании текста в буфер всё корректно — смените шрифт окна консоли вручную или используйте **Windows Terminal**.

Сборка исходников с Unicode-литералами: в проекте включается **`/utf-8`** (см. `Lizard.vcxproj`), иначе строки в `u8` могут собираться неверно на системах с не-UTF-8 кодовой страницей по умолчанию.

### Три вкладки главного меню (1–3)

Клавиши **`1`**, **`2`**, **`3`** переключают **разделы** (активная вкладка помечается `*` в строке табов):

| Клавиша | Раздел | Назначение |
|--------|--------|------------|
| **1** | **Cheats** | Список хуков из `LizardHooks.txt`: **↑ / ↓** — выбор строки, **Enter** — включить/выключить хук (зелёный ON / красный OFF), выбранная строка помечается **`*`**. Если хуков нет, показывается предупреждение. |
| **2** | **Logs** | Просмотр **сессионного лога** (отдельно от вывода меню и логотипа): сообщения резолвера, оффсеты, установка хуков, строки из детуров при срабатывании хуков. Каждая строка с префиксом локального времени **`[YYYY-MM-DD HH:MM:SS]`**. Тот же текст **дополнительно дописывается** в файл **`LizardSession.log`** рядом с `.exe` игры (UTF-8, по строкам). **PgUp / PgDn** — прокрутка по странице. |
| **3** | **Explorer** | Поиск по **`global-metadata.dat` в реальном времени**: ввод **ASCII**-подстроки (без учёта регистра), совпадение проверяется по **имени метода**; **Enter** — выполнить поиск (до **300** совпадений в формате `Namespace.ClassName::MethodName`). **↑ / ↓** — прокрутка списка результатов. Пустой запрос и Enter — подсказка, не поиск. |

При ошибке чтения метаданных в строке статуса показывается **конкретная причина**. Путь к `global-metadata.dat` **переопределяется при каждом поиске** (если `GameAssembly.dll` подгрузился позже — повторите Enter). Рядом с каталогом `GameAssembly.dll` дополнительно ищутся подпапки **`*_Data`** с `il2cpp_data\Metadata\global-metadata.dat`, если имя папки данных не совпадает с базовым именем `.exe`.

**Общее:** **`Esc`** — выход из цикла меню (игра и установленные хуки продолжают работать).

**Замечание:** цифры **1–3** зарезервированы под переключение вкладок; если в поиске в Explorer нужно ввести именно цифру в начале строки, это невозможно без смены раскладки клавиш в коде — для обычных имён методов и ключевых слов используйте буквы.

Реализация: **UI** (табы, баннер, перерисовка) идёт через **`LizardLogf`** только в консоль; **сессионный лог** — **`LizardSessionLogf`** / **`LizardResolverLogf`** / **`LizardLogRawUtf8`** (кольцевой буфер + консоль + файл `LizardSession.log`). Поиск по метаданным — **`metadata_explorer.cpp`** и **`il2cpp_resolver.h`** (`SearchMetadataMethodsForSubstring`).

## Как это работает

1. **Doorstop** подгружает managed bootstrap, тот загружает **`Lizard.dll`**.
2. В отдельном потоке выделяется консоль, читается **`LizardHooks.txt`**, для каждого слота резолвится RVA/оффсет метода через `global-metadata` и **`GameAssembly.dll`**.
3. Ставятся **MinHook**-детуры; при **включённом** пункте меню выполняется логика из конфига; при **выключенном** — вызывается оригинал.
4. **Сессионный лог** (резолвер, хуки, детуры) хранится **отдельно** от меню/логотипа и дублируется в буфер для вкладки **Logs** и в **`LizardSession.log`** рядом с исполняемым файлом; вкладка **Explorer** читает тот же `global-metadata.dat`, что и резолвер, и сканирует таблицу методов.

## Сборка

Вся сборка — скриптом **`build.ps1`**: native DLL, Doorstop bootstrap, копирование в **`.\Release`**, подстановка `coreclr_path` / `corlib_dir` в `doorstop_config.ini`.

### Зависимости

- Visual Studio (MSBuild)
- Git
- vcpkg с пакетом **minhook** (статическая линковка x64)

### Установка зависимостей

1. Установите vcpkg:

   ```text
   git clone https://github.com/Microsoft/vcpkg.git
   cd vcpkg
   .\bootstrap-vcpkg.bat
   .\vcpkg integrate install
   ```

2. Установите minhook:

   ```text
   .\vcpkg install minhook:x64-windows-static
   ```

### Запуск сборки

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

С параметрами:

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1 -Configuration Release -Platform x64
```

Явные пути CoreCLR (если не нужен авто-поиск .NET 6):

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1 `
  -CoreClrPath "C:\Program Files\dotnet\shared\Microsoft.NETCore.App\6.0.xx\coreclr.dll" `
  -CorlibDir "C:\Program Files\dotnet\shared\Microsoft.NETCore.App\6.0.xx"
```

После сборки в **`.\Release`**:

- `Lizard.dll`
- `Doorstop.dll`
- `doorstop_config.ini`
- `winhttp.dll` (из `res\winhttp.dll`)

Секция **`[Il2Cpp]`** в `.\Release\doorstop_config.ini` заполняется скриптом (`coreclr_path`, `corlib_dir`).

Если MSBuild не в PATH, используйте **Developer PowerShell for Visual Studio** или укажите полный путь к `MSBuild.exe`.

## Использование

1. Скопируйте в **корень игры** (рядом с исполняемым файлом) содержимое **`.\Release`**:

   - `winhttp.dll`
   - `doorstop_config.ini`
   - `Doorstop.dll`
   - `Lizard.dll`

2. Положите **`LizardHooks.txt`** рядом с **`.exe` игры`** (или задайте **`LIZARD_HOOKS_FILE`** на полный путь к файлу).

3. Запустите игру. При необходимости отредактируйте хуки и перезапустите игру (горячая перезагрузка конфига в рантайме в текущей версии не описана).

### Mono (ручной инжект)

Для Mono-игр `Lizard.dll` пока инжектируется вручную внешним инжектором. После инжекта функциональность проекта соответствует Il2CPP-режиму (конфиг, меню, runtime-хуки и Explorer-поиск).

## Пример `doorstop_config.ini`

Минимальный ориентир (секция `[Il2Cpp]` после сборки заполняется скриптом):

```ini
[General]
enabled=true
target_assembly=Doorstop.dll
redirect_output_log=false
boot_config_override=
ignore_disable_switch=false

[UnityMono]
dll_search_path_override=
debug_enabled=false
debug_address=127.0.0.1:10000
debug_suspend=false

[Il2Cpp]
coreclr_path=
corlib_dir=
```
